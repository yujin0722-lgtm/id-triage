# 感染症論文トリアージ（Phase 4）

感染症領域の疾患名・病原体名・病態名からPubMed検索式を作成し、検索結果を研究デザイン別に整理して論文選別を支援するウェブアプリです。

## Phase 4で追加した内容

- 現在表示している論文について、全体・未判定・読む・保留・除外の件数を集計
- 「すべて」「未判定」「読む」「保留」「除外」で論文カードを絞り込み
- 絞り込み中に判定を変更した場合、件数と表示をその場で更新
- 各カテゴリーについて、絞り込み後の表示件数を更新
- 「除外」を選択したときだけ除外理由を表示
- 除外理由で「その他」を選択したときだけ自由記載欄を表示
- 自由記載は入力中にブラウザへ保存
- 同じPMIDが別の検索で表示された場合にも、過去の判定・除外理由・自由記載を復元

Phase 3.1までのPubMed検索、研究デザイン分類、論文カード表示はそのまま維持しています。

## GitHub Pagesの更新方法

ZIPを解凍し、`id-paper-triage-phase4`フォルダの中身を、現在のGitHubリポジトリのルートへ上書きアップロードしてください。

```text
assets/
css/
js/
index.html
README.md
id-paper-triage-spec-v0.2.md
```

外側のフォルダではなく、その中身をアップロードします。コミットメッセージの例：

```text
Add Phase 4 screening controls
```

反映後、Windowsでは `Ctrl + F5` で再読み込みしてください。画面右上が `Phase 4` になれば更新されています。

## 動作確認

PubMed検索後、次の順番でご確認ください。

1. 「スクリーニング状況」に全体・未判定・読む・保留・除外の件数が表示される
2. 論文を「読む」にすると、未判定が1件減り、読むが1件増える
3. 「読む」フィルターを押すと、読むと判定した論文だけが表示される
4. 「読む」表示中の論文を「保留」に変えると、そのカードが表示対象から外れる
5. 「除外」を押すと除外理由の選択欄が表示される
6. 除外理由で「その他」を選ぶと自由記載欄が表示される
7. ページを再読み込みして同じ検索を行い、判定・除外理由・自由記載が復元される
8. 過去に判定したPMIDが別の検索結果に含まれた場合も、判定が復元される

## 保存データ

Phase 1〜3.1と同じlocalStorageキーを使うため、既存の検索条件と論文判定は引き継がれます。

- `id-paper-triage:last-search:v1`
- `id-paper-triage:decisions:v1`

「検索条件をクリア」を押しても、論文ごとの判定は削除されません。ブラウザのサイトデータを削除すると保存内容も消えます。

## 研究デザイン分類について

研究デザインはPubMedから取得したPublication Typeだけに基づいて分類します。タイトルや抄録から推測せず、論文の質、エビデンスの確実性、推奨の強さも評価しません。

## ファイル名について

文字化けを避けるため、配布ファイル名と仕様書ファイル名は半角英数字にしています。文書内のタイトルと本文は日本語です。
